# Contributing

Thanks for helping improve YTLite.

1. Open an issue before starting a large behavior or interface change.
2. Create a focused branch and keep unrelated changes separate.
3. Run `xcodegen generate` and the full `xcodebuild ... test` command from the README.
4. Explain user-visible behavior and verification in the pull request.

Please never include authentication cookies, account data, private download URLs, downloaded media, signing certificates, or Apple notarization credentials in an issue or commit.
